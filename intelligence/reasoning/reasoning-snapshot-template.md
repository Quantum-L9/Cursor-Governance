# Reasoning Snapshot Template

## Purpose
This template provides a structured format for documenting governance decisions and reasoning processes within Suite 6. It integrates Constellation's reasoning snapshot format with Suite 6's 10-step reasoning framework.

## Template Structure

### Basic Reasoning Snapshot Format
```markdown
### Reasoning Snapshot
**Agent/Component**: [Component ID and Name]
**Request/Decision**: [What decision was being made]
**Rule Applied**: [FOL rule from rule registry]
**Context**: [Relevant context and constraints]
**Observation**: [What was observed/analyzed]
**Conclusion**: [Reasoning conclusion]
**Verdict**: [Y/N or Pass/Fail]
**Timestamp**: [ISO 8601 timestamp]
**Snapshot ID**: [Unique identifier]
```

### Enhanced Suite 6 Format
```markdown
### Suite 6 Reasoning Snapshot

#### Decision Context
- **Component**: [Component ID] - [Component Name]
- **Decision Type**: [governance_validation | rule_enforcement | escalation_routing | other]
- **Initiated By**: [System | User | Agent]
- **Priority Level**: [critical | high | medium | low]
- **Timestamp**: [2025-10-28T12:00:00Z]

#### Governance Framework
- **Rule Applied**: [FOL rule from registry]
- **Rule ID**: [R001, R002, etc.]
- **Kernel Version**: [6.0]
- **Escalation Path**: [Roundtable | Chair_Override | HumanOperator | Auto_Snapshot]

#### 10-Step Reasoning Process
1. **Objective**: [Clear problem statement]
2. **Context**: [Hydrated context and constraints]
3. **System Analysis**: [Component relationships and dependencies]
4. **Strategy**: [Chosen approach and rationale]
5. **Analysis**: [Detailed evaluation and edge cases]
6. **Decision**: [Clear recommendation with rationale]
7. **Validation**: [Approach verification and testing]
8. **Documentation**: [Implementation details and requirements]
9. **Risk Assessment**: [Identified risks and mitigation]
10. **Maintainability**: [Long-term considerations]

#### Observation & Evidence
- **Data Sources**: [List of information sources]
- **Key Findings**: [Critical observations]
- **Constraints**: [Limitations and boundaries]
- **Assumptions**: [Underlying assumptions made]

#### Decision Outcome
- **Conclusion**: [Detailed reasoning conclusion]
- **Verdict**: [PASS | FAIL | ESCALATE | DEFER]
- **Confidence Level**: [High | Medium | Low]
- **Next Actions**: [Required follow-up actions]

#### Audit Trail
- **Snapshot ID**: [Unique identifier: SNAP-{timestamp}-{component}]
- **Generated By**: [Suite 6 component or agent]
- **Validation Status**: [Validated | Pending | Failed]
- **Digital Signature**: [SHA256 hash for integrity]
- **Related Snapshots**: [Links to related decisions]
```

## Integration Points

### With Cursor Native Reasoning
- Snapshots complement the 10-step reasoning framework
- Provide structured documentation for each reasoning step
- Enable audit trail for complex decisions

### With Governance Validation
- Automatic snapshot generation for validation failures
- Integration with escalation routing system
- Compliance tracking and reporting

### With Digital Signatures
- Each snapshot is digitally signed for integrity
- Tamper detection for audit trail protection
- Verification of snapshot authenticity

## Automation Guidelines

### Automatic Snapshot Generation
Snapshots should be automatically generated for:
- Governance rule violations
- Escalation routing decisions
- Critical component validations
- Security-related decisions
- Performance threshold breaches

### Manual Snapshot Creation
Manual snapshots recommended for:
- Complex architectural decisions
- Policy changes and updates
- Strategic governance modifications
- Cross-component integration decisions

## Validation Requirements

### Required Fields
All snapshots must include:
- Component identification
- Decision context and type
- Applied governance rules
- Reasoning process documentation
- Clear verdict and next actions
- Audit trail information

### Quality Standards
- Clear, concise language
- Traceable decision logic
- Complete evidence documentation
- Proper governance rule references
- Valid digital signatures

- Compliance reporting integration
